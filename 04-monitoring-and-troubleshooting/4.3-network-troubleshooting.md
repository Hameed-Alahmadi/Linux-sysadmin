## 🔌 Socket and Listener Auditing

Inspect loaded ports, active TCP sockets, UDP listeners, and associated process identities.

```bash
# List all active listening TCP and UDP ports with process identities
sudo ss -tulpn

# Classic alternative command listing active listeners and TCP connections
sudo netstat -tulpn

# Map open file indicators and network sockets back to active processes
sudo lsof -i :80
```

---

## 🌐 External Connectivity Validation

Inspect end-to-end interface connections, DNS mapping profiles, and routing hop paths.

```bash
# Send ICMP echo requests to verify raw target host reachability
ping -c 4 1.1.1.1

# Trace the exact routing path hops taken to reach a remote server
traceroute 1.1.1.1

# Interrogate external nameservers to verify correct DNS domain configurations
dig google.com

# Confirm simple name resolution answers quickly
nslookup google.com
```

---

## 🧪 Application Layer Testing (Telnet / Curl / Ncat)

Test remote port bindings, request HTTP headers, and manually interact with network ports.

```bash
# Test raw TCP handshakes to confirm if a specific port is open
# (If successful, connection opens; hit Ctrl+Click or Ctrl+] to escape)
telnet 192.168.1.150 80

# Retrieve remote web server HTTP headers without loading the full body payload
curl -I [https://www.google.com](https://www.google.com)

# Create an on-demand TCP link listener on a local port for packet validation tests
nc -l 8080
```

## 📦 Cross-Distribution Package Operations

Manage system software packages safely. Always run updates and simulation checks before installing new utilities on a production host.

```bash
# Update local package indexes and upgrade all system packages (Ubuntu/Debian)
sudo apt update && sudo apt upgrade -y

# Perform a dry-run install simulation to audit dependencies without installing (Ubuntu/Debian)
sudo apt install --dry-run nginx

# Search for packages across remote repository databases
apt search apache2

# Remove a package but preserve its global configuration files
sudo apt remove nginx

# Purge a package completely along with all associated configuration layouts
sudo apt purge nginx
```

## ⚙️ Systemd Service Management (`systemctl`)

Control background system daemons, check state parameters, and configure startup rules.

```bash
# Query the precise runtime status of a service daemon (e.g., Nginx)
sudo systemctl status nginx

# Start an inactive background daemon instantly
sudo systemctl start nginx

# Stop a running system daemon
sudo systemctl stop nginx

# Restart a daemon (causes a brief connection interruption)
sudo systemctl restart nginx

# Reload a daemon to read configuration changes without terminating connection states
sudo systemctl reload nginx

# Configure a daemon to launch automatically during system initialization
sudo systemctl enable nginx

# Prevent a service from launching automatically during system initialization
sudo systemctl disable nginx

# Check whether a specific service is currently configured to run at boot
sudo systemctl is-enabled nginx
```

## 🔍 Log Auditing with `journalctl`

Troubleshoot service failures and query background log outputs using systemd's built-in logging utility.

```bash
# Output log histories for a specific service target in reverse-chronological order
journalctl -u nginx -r

# Stream real-time logs for a specific service as they are appended to the system
journalctl -u nginx -f

# Filter system logs to display errors and critical messages only
journalctl -p err..emerg

# Show all system logs generated since the current system boot sequence
journalctl -b
```

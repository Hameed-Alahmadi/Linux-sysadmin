## 🔍 Storage Space and Block Device Auditing

Analyze mounted filesystems, directory storage consumption, and available raw block devices.

```bash
# List all available block devices, partition tables, and mount points
lsblk

# Display free and used disk space on all currently mounted filesystems (human-readable)
df -h

# Estimate and display space usage of a specific directory and its subdirectories
du -sh /var/log

# List partition tables for all active block devices with admin privileges
sudo fdisk -l
```

---

## 🛠️ Disk Partitioning and Formatting

Prepare a raw disk block device for operation by partitioning and writing a clean filesystem.

```bash
# Partition a raw disk device (e.g., /dev/sdb) using fdisk interactive utility
# sudo fdisk /dev/sdb

# Write a clean, modern Ext4 filesystem to a target partition
sudo mkfs.ext4 /dev/sdb1

# Write an enterprise-grade XFS filesystem to a target partition
sudo mkfs.xfs /dev/sdb2
```

## 💾 Mount Operations and Persistent Configuration

Mount the filesystem manually for immediate access, and then make it persistent across system restarts.

```bash
# Create a dedicated directory to act as your target mount point
sudo mkdir -p /mnt/data

# Mount the partition to the target mount point manually
sudo mount /dev/sdb1 /mnt/data

# Retrieve the unique UUID of your partition for safe persistent mounting
sudo blkid /dev/sdb1

# Configuration template inside /etc/fstab for persistent mounting:
# UUID=YOUR-UUID-HERE    /mnt/data    ext4    defaults    0    2

# Reload the systemd daemon and mount all points declared inside /etc/fstab
sudo systemctl daemon-reload
sudo mount -a

# Unmount the filesystem when it is no longer actively in use
sudo umount /mnt/data
```

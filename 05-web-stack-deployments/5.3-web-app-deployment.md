## 🛠️ Preparing the Web App Storage Directory

Download the latest release distribution and set up directories with safe Unix permissions.

```bash
# Create the deployment directory for your application
sudo mkdir -p /var/www/yourdomain

# Retrieve and extract the latest WordPress release package
cd /tmp
wget [https://wordpress.org/latest.tar.gz](https://wordpress.org/latest.tar.gz)
tar -xzf latest.tar.gz

# Move raw package assets to the production virtual host root folder
sudo cp -r wordpress/* /var/www/yourdomain/
```

---

## 👑 Assigning Safe Ownership and Permissions

Configure access policies so that your web host daemon (typically running as user `www-data` on Debian/Ubuntu) can read files and write updates without excessive privileges.

```bash
# Set owner and group permissions for the directories to the web service user
sudo chown -R www-data:www-data /var/www/yourdomain/

# Enforce standard, secure directory permissions (rwxr-xr-x)
sudo find /var/www/yourdomain/ -type d -exec chmod 755 {} \;

# Enforce standard, secure file permissions (rw-r--r--)
sudo find /var/www/yourdomain/ -type f -exec chmod 644 {} \;
```

## ⚙️ App Configuration and DB Linkage

Define the application parameters and link it directly to your secured database container.

```bash
# Create the local application configuration file using the default template
cd /var/www/yourdomain/
cp wp-config-sample.php wp-config.php

# Recommended updates inside the wp-config.php file:
# define( 'DB_NAME', 'webapp_db' );
# define( 'DB_USER', 'webapp_user' );
# define( 'DB_PASSWORD', 'Hardened_Password_Here' );
# define( 'DB_HOST', 'localhost' );
```

## 🛡️ Generating SSL Credentials via Certbot

The ACME toolset automates securing domains with verified, signed certificates from the Let’s Encrypt authority.

```bash
# Install certbot package utilities and web server plugins
sudo apt update
sudo apt install certbot python3-certbot-nginx python3-certbot-apache -y

# Automated SSL setup for Nginx (installs certs and configures system blocks automatically)
sudo certbot --nginx -d yourdomain.com -d [www.yourdomain.com](https://www.yourdomain.com)

# Automated SSL setup for Apache (installs certs and configures system virtual hosts automatically)
sudo certbot --apache -d yourdomain.com -d [www.yourdomain.com](https://www.yourdomain.com)
```

## ⚙️ Certbot Renewals and Configuration Testing

Let's Encrypt certificates are valid for 90 days. Confirm the automated background cron renewal process is working properly.

```bash
# Perform a dry-run renewal simulation to test system stability
sudo certbot renew --dry-run
```

## 🔒 Hardening Server TLS Parameters

Mitigate security vulnerabilities like POODLE and BEAST by disabling weak TLS protocols and legacy ciphers.

```bash
# Recommended SSL Hardening rules inside Nginx server block files:
# ssl_protocols TLSv1.2 TLSv1.3;
# ssl_prefer_server_ciphers on;
# ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384';

# Reload your web server daemon to bring security parameters online
sudo nginx -t && sudo systemctl reload nginx
```

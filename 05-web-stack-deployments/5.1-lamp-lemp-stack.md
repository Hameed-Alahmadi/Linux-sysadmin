## 🌐 Apache Web Stack (LAMP Setup)

LAMP constitutes a combined Linux, Apache web host, MySQL engine database, and PHP process runtime environment.

```bash
# Install the core LAMP packages
sudo apt update
sudo apt install apache2 mariadb-server php libapache2-mod-php php-mysql -y

# Configure Apache virtual hosts for custom domains
# Configuration path target: /etc/apache2/sites-available/yourdomain.conf
# <VirtualHost *:80>
#     ServerAdmin admin@yourdomain.com
#     ServerName yourdomain.com
#     ServerAlias [www.yourdomain.com](https://www.yourdomain.com)
#     DocumentRoot /var/www/yourdomain
#     ErrorLog ${APACHE_LOG_DIR}/error.log
#     CustomLog ${APACHE_LOG_DIR}/access.log combined
# </VirtualHost>

# Enable your virtual host configuration and reload the service
sudo a2ensite yourdomain.conf
sudo a2dissite 000-default.conf
sudo systemctl reload apache2
```

## ⚡ Nginx Web Stack (LEMP Setup)

LEMP utilizes Nginx as the reverse proxy/web host and relies on php-fpm (PHP FastCGI Process Manager) to handle PHP processing.

```bash
# Install the core LEMP packages
sudo apt install nginx mariadb-server php-fpm php-mysql -y

# Configure an Nginx Server Block
# Configuration path target: /etc/nginx/sites-available/yourdomain
# server {
#     listen 80;
#     server_name yourdomain.com [www.yourdomain.com](https://www.yourdomain.com);
#     root /var/www/yourdomain;
#     index index.php index.html index.htm;
#
#     location / {
#         try_files $uri $uri/ =404;
#     }
#
#     location ~ \.php$ {
#         include snippets/fastcgi-php.conf;
#         fastcgi_pass unix:/var/run/php/php-fpm.sock;
#     }
# }

# Enable the server block and restart Nginx
sudo ln -s /etc/nginx/sites-available/yourdomain /etc/nginx/sites-enabled/
sudo systemctl restart nginx
```

## 🗄️ Database Provisioning & Hardening

Install, lock down, and provision secure schemas inside the database system.

```bash
# Secure the initial database instance (deletes test tables, anonymous accounts, and locks root access)
sudo mysql_secure_installation

# Log into the database command console as administrative root
sudo mysql -u root -p

# Secure SQL database creation commands:
# CREATE DATABASE webapp_db;
# CREATE USER 'webapp_user'@'localhost' IDENTIFIED BY 'Hardened_Password_Here';
# GRANT ALL PRIVILEGES ON webapp_db.* TO 'webapp_user'@'localhost';
# FLUSH PRIVILEGES;
# EXIT;
```

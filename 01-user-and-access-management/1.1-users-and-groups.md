## 👤 Core Concepts: Creating and Modifying Users

Linux accounts can be created or altered via low-level native utilities. 

```bash
# Create a new user with a home directory, custom shell, and specific secondary groups
sudo useradd -m -G sudo,games -s /bin/bash <userName>

# Create a user with a specific comment (metadata) and no matching private group
sudo useradd -N -G sudo -c "Jude Bellingham is The Best" -s /bin/bash jude

```

### Essential CLI Flags for Account Manipulation

When configuring accounts with `useradd` or updating existing accounts via `usermod`, use the following flag map:

| Option | Functional Purpose | Contextual Example |
| --- | --- | --- |
| `-m` | Forces the creation of the user's home directory space | `useradd -m staticuser` |
| `-N` | Prevents the creation of a private group matching the username | `useradd -N staticuser` |
| `-g <group>` | Overrides or assigns the user's Primary Group | `usermod -g developers alex` |
| `-G <group>` | Overrides or assigns supplemental/complementary group memberships | `usermod -G games,sudo,docker alex` |
| `-aG <group>` | Appends complementary groups safely without overwriting current states | `usermod -aG sysadmin jude` |
| `-c "text"` | Appends system metadata, definitions, or descriptive details/comments | `usermod -c "The Real GOAT" jude` |
| `-s <shell>` | Sets the default operational login shell execution environment | `useradd -s /bin/bash jude` |
| `-d <dir>` | Modifies the user's landing/home directory target pointer path | `usermod -d /var/www/jude jude` |
| `-e <date>` | Schedules a definitive account expiration threshold date (`YYYY-MM-DD`) | `usermod -e 2026-12-31 alex` |
| `-l <name>` | Changes the account login name string | `usermod -l newUserName currentUserName` |

### Account Locking Protocols

```bash
# Temporarily lock a user account out of the system
sudo usermod -L <userName>

# Unlock the locked account to resume active entry access
sudo usermod -U <userName>

```

## ⚙️ System Policy & Aging Configuration Files

To establish global, system-wide constraints on newly provisioned user accounts, modify these underlying file templates:

### 1. Global Password Rules: `/etc/login.defs`

Controls default shadow password suites, min/max usage day metrics, and warning timeframes before forcing a credential update:

```text
PASS_MAX_DAYS   99999   # Maximum number of days a password may be used
PASS_MIN_DAYS   0       # Minimum number of days allowed between password changes
PASS_WARN_AGE   7       # Number of days warning given before a password expires

```

### 2. Provisioning Blueprints: `/etc/default/useradd`

Defines fallback shells, home folder root paths, skeletal configuration templates, and post-expiration states:

```text
SHELL=/bin/sh           # Default shell if unspecified
GROUP=100               # Default fallback user GID mapping
HOME=/home              # Base tree target path for user home locations
INACTIVE=-1             # Account grace period days after password expiration (-1 disables it)
SKEL=/etc/skel          # Source directory containing system-wide default profiles

```

> 💡 **Note**: If you alter the default `INACTIVE=-1` to a value like `INACTIVE=14`, the user is granted exactly 14 days of continued workspace access after password expiration to resolve credentials before the system locks the account completely.

## 🔑 Interactive & Non-Interactive Password Management

### Interactive Method (Standard)

```bash
sudo passwd <userName>

```

### Non-Interactive Methods (Automation & Provisioning Scripts)

Highly efficient for system administration automation tools and batch environment setups to prevent terminal prompts:

```bash
# Red Hat / Fedora Distros (Using stdin streaming standard)
echo "<newPassword>" | sudo passwd <userName> --stdin

# Debian / Ubuntu / Arch Distros (Direct processing map stream)
echo "<userName>:<newPassword>" | sudo chpasswd

```

## ⏳ The `chage` Utility (Password Aging Management)

The `chage` utility modifies the exact number of days between password changes, expiration warnings, and account block windows.

```bash
# General syntax layout
chage <options> <userName>

# Example: Enforce password changes every 10 days, allow immediate changes, and disable account expiration thresholds
sudo chage -M 10 -m 0 -E -1 test

```

### Reference Flags for `chage`

* `-m`: Set the minimum number of days before a password change is allowed.
* `-M`: Set the maximum number of days a password remains valid.
* `-W`: Set the number of warning days given before a password expires.
* `-I`: Set password inactivity periods before completely disabling access.
* `-E`: Set a fixed account expiration date (`-1` to disable).
* `-l`: Inspect active age limitations and account policies.
* `-h`: Display help options and formatting syntax.

## 👥 Group Administration

Groups organize system permissions into shared, manageable organizational units.

```bash
# Instantiate a completely fresh user access group mapping
sudo groupadd <groupName>

# Append an existing system user account into a supplemental group
sudo usermod -aG <groupName> <userName>
# OR alternative tool assignment syntax:
sudo gpasswd -a <userName> <groupName>

# Strip a specific user's membership away from an individual group
sudo gpasswd -d <userName> <groupName>

# Rename an existing group designation string
sudo groupmod -n <newGroupName> <currentGroupName>

# Permanently destroy a group allocation rule
sudo groupdel <groupName>

```

## 🔍 System Audit & Integrity Inspection Commands

Verify system integrity, interrogate core tables, and clean up orphaned file assets:

```bash
# View active real-time details of an existing profile (UID, GID, Groups)
id <userName>

# Interrogate structural flat file configurations for confirmation
getent passwd <userName>                  # Query details directly out of /etc/passwd
getent group | grep <groupName>           # Query detailed listings of explicit group maps
cat /etc/group                            # Check a raw snapshot dump of all group definitions

```

### Account Deletion and Orphan File Audits

```bash
# Soft delete user (removes entry from pass tables but leaves home folder assets intact)
sudo userdel <userName>

# Hard delete (removes accounts and completely destroys user home directory workspaces)
sudo userdel -r <userName>

# Force hard delete (forces a complete user purge even if files are open or user is busy)
sudo userdel -rf <userName>

# Alternative manual home wipe if a soft delete was run previously
rm -rf /home/<userName>

# Audit/Locate unowned files across volumes (orphaned assets with no valid user associated)
find /home -nouser            # Search within local user home environments
find /var -nouser             # Search within volatile system log/data spaces
find /home /var -nouser       # Scan multiple high-activity trees concurrently

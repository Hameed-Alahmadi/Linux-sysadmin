# 01 - User & Access Management: Access Control Lists (ACL)

This reference module explains how to implement fine-grained file and directory access rules using Access Control Lists (ACLs). Standard Linux permissions (`rwx` for User, Group, and Others) can be restrictive in multi-user settings; ACLs allow you to grant specific permissions to individual users or groups without altering core ownership mappings.

## 🔍 Pre-Requisite Verification

Before manipulating ACL permissions, ensure your system kernel includes support for access rules. You can verify this by checking your active boot configuration file:

```bash
grep -i acl /boot/config-$(uname -r)

```

---

## 🛠️ Interrogating and Setting ACLs

### 1. View Active Permissions Matrix

To check the specialized permissions assigned to a target file or folder, use `getfacl`:

```bash
getfacl <folderName>

```

### 2. Modifying and Appending Rule Blocks (`setfacl`)

The basic syntax allows you to target either a user (`u`) or a group (`g`):

```bash
# Add/Modify specific user access permissions to a directory
sudo setfacl -m u:<userName>:rwx <folderName>

# Add/Modify specific group access permissions to a directory
sudo setfacl -m g:<groupName>:rx <folderName>

```

### Essential CLI Flags for ACL Manipulation

When configuring or altering access control settings with `setfacl`, use the following flag map:

| Option | Functional Purpose | Contextual Example |
| --- | --- | --- |
| `-m` | Modifies or adds new ACL rules to the specified target asset | `setfacl -m u:alex:rw file.txt` |
| `-x` | Removes specific ACL rules from an individual user or group | `setfacl -x u:alex file.txt` |
| `-b` | Removes all extended ACL rules completely, resetting to standard permissions | `setfacl -b file.txt` |
| `-R` | Recursively applies ACL rules down into all files and subdirectories | `setfacl -R -m g:devs:rx folder/` |
| `-d` | Sets default ACL entries (only works on directories) for future assets | `setfacl -m d:u:alex:rwx folder/` |
| `-k` | Removes all default ACL settings from a directory asset | `setfacl -k folder/` |

## ❌ Removing and Resetting Rule Vectors

### 1. Stripping Individual Rules Safely

If you want to revoke access rules for a specific entity without modifying other existing parameters:

```bash
# Revoke a specific user's ACL ruleset
sudo setfacl -x u:<userName> <folderName>

# Revoke a default rule mapping for a specific user
sudo setfacl -x d:u:<userName> <folderName>

```

### 2. Purging All Extended Rules Consistently

To completely strip away all customized ACL configurations and restore standard system permissions:

```bash
setfacl -b <folderName>

```

---

## 💾 Backup and Recovery Workflows

When migrating workspaces or preserving configurations before hardware changes, you can back up and completely restore the full permissions matrix of an entire directory tree.

```bash
# Export permissions recursing all target files and folders
getfacl -R /test > /tmp/acl_backup.bak

```

> ⚠️ **Important Deployment Rule**: When restoring, you must be in the parent directory of the original target folder workspace so relative system indexing maps back over cleanly.

```bash
# Restore entire permissions schema cleanly from the backup file
setfacl --restore=/tmp/acl_backup.bak

```

# 01 - User & Access Management: Access Control Lists (ACL)

This reference module explains how to implement fine-grained file and directory access rules using Access Control Lists (ACLs). Standard Linux permissions (`rwx` for User, Group, and Others) can be restrictive in multi-user settings; ACLs allow you to grant specific permissions to individual users or groups without altering core ownership mappings.

## 🔍 Pre-Requisite Verification

Before manipulating ACL permissions, ensure your system kernel includes support for access rules. You can verify this by checking your active boot configuration file:

```bash
grep -i acl /boot/config-$(uname -r)
🛠️ Interrogating and Setting ACLs1. View Active Permissions MatrixTo check the specialized permissions assigned to a target file or folder, use getfacl:Bashgetfacl <folderName>
2. Modifying and Appending Rule Blocks (setfacl)The basic syntax allows you to target either a user (u) or a group (g):Bash# Add/Modify specific user access permissions to a directory
sudo setfacl -m u:<userName>:rwx <folderName>

# Add/Modify specific group access permissions to a directory
sudo setfacl -m g:<groupName>:rx <folderName>
Essential CLI Flags for ACL ManipulationWhen configuring or altering access control settings with setfacl, use the following flag map:OptionFunctional PurposeContextual Example-mModifies or adds new ACL rules to the specified target assetsetfacl -m u:alex:rw file.txt-xRemoves specific ACL rules from an individual user or groupsetfacl -x u:alex file.txt-bRemoves all extended ACL rules completely, resetting to standard permissionssetfacl -b file.txt-RRecursively applies ACL rules down into all files and subdirectoriessetfacl -R -m g:devs:rx folder/-dSets default ACL entries (only works on directories) for future assetssetfacl -m d:u:alex:rwx folder/-kRemoves all default ACL settings from a directory assetsetfacl -k folder/❌ Removing and Resetting Rule Vectors1. Stripping Individual Rules SafelyIf you want to revoke access rules for a specific entity without modifying other existing parameters:Bash# Revoke a specific user's ACL ruleset
sudo setfacl -x u:<userName> <folderName>

# Revoke a default rule mapping for a specific user
sudo setfacl -x d:u:<userName> <folderName>
2. Purging All Extended Rules ConsistentlyTo completely strip away all customized ACL configurations and restore standard system permissions:Bashsetfacl -b <folderName>
💾 Backup and Recovery WorkflowsWhen migrating workspaces or preserving configurations before hardware changes, you can back up and completely restore the full permissions matrix of an entire directory tree.Bash# Export permissions recursing all target files and folders
getfacl -R /test > /tmp/acl_backup.bak
⚠️ Important Deployment Rule: When restoring, you must be in the parent directory of the original target folder workspace so relative system indexing maps back over cleanly.Bash# Restore entire permissions schema cleanly from the backup file
setfacl --restore=/tmp/acl_backup.bak

## 🔑 Key Authentication Setup

Configuring public-private key pair authentication is the single most critical step to secure SSH logins against brute-force attacks.

```bash
# Generate a high-security asymmetric RSA key pair on your local client machine
ssh-keygen -t rsa -b 4096

# Transfer your public key securely to the remote target host
# (Replace user and server_ip with your target credentials)
ssh-copy-id -i ~/.ssh/id_rsa.pub user@server_ip
```

## 🔒 Configuration Hardening (/etc/ssh/sshd_config)

To harden the daemon, edit the primary configuration file located at `/etc/ssh/sshd_config`. Standardize your configuration parameters with the recommended values displayed below.

```bash
# Edit the configuration file with administrative privileges
# sudo nano /etc/ssh/sshd_config

# Recommended secure configuration values inside /etc/ssh/sshd_config:
# Port 2222                 # Change the default port 22 to reduce background scans
# PermitRootLogin no        # Strictly disable root-level login access over SSH
# PasswordAuthentication no  # Disable standard password prompts entirely
# PubkeyAuthentication yes  # Enforce SSH key authentication only
# MaxAuthTries 3            # Limit authentication attempts to mitigate brute-force loops
```

## ⚙️ Service Control and Testing

Apply configuration shifts safely and verify daemon operability without severing existing SSH sessions.

```bash
# Validate your configuration syntax for errors before restarting
sudo sshd -t

# Apply your secure adjustments by restarting the SSH background service
sudo systemctl restart sshd

# Query active process states to verify the daemon is healthy
sudo systemctl status sshd
```

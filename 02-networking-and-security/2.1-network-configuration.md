## 🔍 Interface Diagnostics and State Auditing

Investigate network hardware, layer-3 addressing profiles, and operational interface states.

```bash
# Display all operational network interfaces along with assigned IP addresses
ip address show

# Display a summarized, brief translation of active interface properties
ip -br address show

# Check link-layer operational states and carrier detection logs
ip link show

# Check hardware transceiver statistics, driver attributes, and link speeds
sudo ethtool eth0
⚙️ Persistent Configuration Engines
Modern Linux distributions manage persistent interface configurations through either systemd-networkd or netplan.

1. Systemd-Networkd Configurations
Commonly deployed on minimalist server instances and Arch Linux systems. Configuration layouts reside within /etc/systemd/network/.

Example DHCP configuration file: /etc/systemd/network/20-wired.network
Ini, TOML
[Match]
Name=enp3s0

[Network]
DHCP=yes
Example Static IP configuration file: /etc/systemd/network/10-static-enp3s0.network
Ini, TOML
[Match]
Name=enp3s0

[Network]
Address=192.168.1.150/24
Gateway=192.168.1.1
DNS=1.1.1.1
DNS=8.8.8.8
Bash
# Apply adjustments by restarting the background link manager service
sudo systemctl restart systemd-networkd
2. Netplan Declarative Engine
Standard deployment controller for Ubuntu environments. Configurations are formatted using YAML syntax.

Example layout schema: /etc/netplan/01-netcfg.yaml
YAML
network:
  version: 2
  renderer: networkd
  ethernets:
    enp3s0:
      dhcp4: no
      addresses:
        - 192.168.1.150/24
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        addresses: [1.1.1.1, 8.8.8.8]
Bash
# Validate config parameters for errors before deployment
sudo netplan try

# Force apply the running configurations immediately
sudo netplan apply
🛣️ Core Routing Manipulation
Inspect, adjust, or completely revoke localized target routing maps.

Bash
# Interrogate the current IP routing table architecture
ip route show

# Establish a temporary manual default system gateway endpoint
sudo ip route add default via 192.168.1.1 dev enp3s0

# Explicitly prune a specialized target route tracking out of memory
sudo ip route del 10.0.0.0/8 via 192.168.1.254
🌐 Name Resolution Controls (/etc/resolv.conf)
The primary file engine mapping local target domain queries to external DNS name resolvers.

Plaintext
nameserver 1.1.1.1
nameserver 8.8.8.8
search localdomain
⚠️ Systemd-Resolved Notice: On modern installations, /etc/resolv.conf is typically a managed symbolic link pointing back to systemd-resolved. Avoid editing this file manually; utilize your network renderer's settings or adjust configuration properties directly within /etc/systemd/resolved.conf.

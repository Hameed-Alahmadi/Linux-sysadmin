# 02 - Networking & Security: Firewall Infrastructure (UFW)

This reference guide covers the implementation of host-based packet filtering using the Uncomplicated Firewall (UFW) wrapper, setting secure default policies, and configuring rules for application services.

## 🔍 Status Verification and Initialization

Before enabling the firewall daemon, inspect its current status and running configurations.

```bash
# Check the status of UFW (active, inactive, and loaded rules)
sudo ufw status verbose

# Display loaded rules numbered sequentially (useful for selective deletion)
sudo ufw status numbered
```

---

## 🛡️ Setting Default Security Policies

Establish a strong default security posture. Block all incoming attempts by default while allowing outbound traffic.

```bash
# Set default behavior to deny all incoming traffic
sudo ufw default deny incoming

# Set default behavior to allow all outgoing traffic
sudo ufw default allow outgoing
```

---

## ⚙️ Rule Configuration for Services

Explicitly permit traffic to your system services by port number, protocol, or application profile.

```bash
# Allow SSH traffic on default port 22
sudo ufw allow 22/tcp

# Allow SSH traffic on a custom hardened port (matching your sshd_config)
sudo ufw allow 2222/tcp

# Allow HTTP and HTTPS web traffic (ports 80 and 443)
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# Allow traffic coming from a specific trusted subnet or IP address
sudo ufw allow from 192.168.1.0/24 to any port 22 proto tcp
```

---

## 🔒 Advanced Filtering and Rate Limiting

Mitigate brute-force access attempts on critical endpoints using built-in connection limiting.

```bash
# Throttle connection attempts on port 22 (limits IPs with 6+ attempts in 30s)
sudo ufw limit 22/tcp

# Delete a rule by referring to its specific number (obtained from ufw status numbered)
# sudo ufw delete 3
```

---

## 🚀 Activating and Reloading the Firewall

Safely bring the ruleset online. Ensure you have allowed your active SSH port before enabling to prevent lockout.

```bash
# Enable the firewall service and configure it to start automatically on system boot
sudo ufw enable

# Apply new configuration changes without disrupting existing connections
sudo ufw reload

# Disable the firewall and suspend all running packet filtering rules
sudo ufw disable
```
